Database Overview

  • Table: markets
  • Primary Key: listing_id (INTEGER)
  • Sample Columns: update_time (TEXT), listing_name (TEXT), location_address (TEXT), location_x (REAL), location_y (REAL), specialproductionmethods (TEXT), FNAP (TEXT)
  • Data: Multi-value fields (semicolon-separated in FNAP and specialproductionmethods)

Explanation: The USDA dataset contains ~7,000 farmers markets with location, programs accepted (FNAP), production methods, and more. Multi-value fields require string pattern matching (LIKE '%value%').

DDL Basic: Creating the Table

CREATE TABLE markets ( listing_id INTEGER PRIMARY KEY, update_time TEXT, listing_name TEXT, location_address TEXT, location_x REAL, location_y REAL, specialproductionmethods TEXT );

Explanation: Creates the base table structure. Primary key ensures unique market IDs. TEXT for strings, REAL for coordinates.

Intermediate DDL: Indexing for Performance

CREATE INDEX idx_listing_name ON markets(listing_name);

Explanation: Speeds up searches and sorts on listing_name (e.g., WHERE listing_name LIKE '%Farm%').

Advanced DDL: Adding Constraints

ALTER TABLE markets ADD COLUMN organic_flag INTEGER CHECK (organic_flag IN (0,1));

Explanation: Enforces that organic_flag can only be 0 or 1 (boolean-like). Prevents invalid data entry.

Advanced DQL: Markets per Indoor Type

SELECT location_indoor, COUNT(*) AS count FROM markets GROUP BY location_indoor;

Advanced DQL: Compare Coordinates (Self-Join)

SELECT m1.listing_name, m2.listing_name FROM markets m1 JOIN markets m2 ON m1.listing_id < m2.listing_id WHERE ABS(m1.location_y - m2.location_y) < 0.1;

Explanation: Finds pairs of markets close in latitude (simple proxy for nearby).

Exercise: Modify to find markets within 0.2 degrees latitude AND longitude of each other.

Advanced DQL: Markets Updated in 2026 (Subquery)

SELECT listing_name FROM markets WHERE update_time IN ( SELECT update_time FROM markets WHERE strftime('%Y', update_time) = '2026' );

Explanation: Uses subquery to find records from the same update batch as 2026 entries.

Exercise: Find markets whose update_time is earlier than the most recent update_time in the table.

Advanced DQL: Combine Queries (UNION)

SELECT listing_name FROM markets WHERE FNAP LIKE '%SNAP%' UNION SELECT listing_name FROM markets WHERE FNAP LIKE '%WIC%';

Explanation: Gets distinct list of markets accepting either SNAP or WIC.

Exercise: Use UNION ALL (with duplicates) to see how many markets accept both programs (count overlaps).
